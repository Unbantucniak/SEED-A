# 实验验证方案设计
## 第四阶段核心交付物

### 1. 实验目标
验证本项目提出的自学习自演化经验积累方法的有效性，对比不同基线方案在典型开发任务场景下的性能差异，量化评估方法的实际收益。

### 2. 评测任务场景
选取3类典型软件开发任务作为评测场景：
| 任务类型 | 任务描述 | 评测指标 | 任务数量 |
|----------|----------|----------|----------|
| **代码生成** | 根据自然语言需求生成对应功能的Python/Java代码 | 代码正确率、通过测试用例比例、生成耗时、交互轮次 | 50个（覆盖算法、业务逻辑、工具类等场景） |
| **bug修复** | 给定存在bug的代码和bug描述，修复代码问题 | 修复成功率、修复耗时、代码修改量 | 30个（覆盖语法错误、逻辑错误、性能问题等） |
| **需求分解** | 给定复杂项目需求，分解为可执行的子任务 | 分解合理性、子任务完整性、耗时 | 20个（覆盖中小规模项目需求） |

### 3. 对比基线方案
选择4种业界常用方案作为对比基线：
| 基线编号 | 方案名称 | 方案描述 |
|----------|----------|----------|
| **Baseline 1** | 无经验积累 | 普通大模型直接调用，无任何记忆/经验模块 |
| **Baseline 2** | 仅向量检索（RAG） | 仅使用向量数据库存储历史交互，检索相关上下文作为prompt补充 |
| **Baseline 3** | 周期性微调 | 每100个任务对小模型进行一次全量微调，无RAG模块 |
| **Baseline 4** | 商用开发智能体 | 对比GitHub Copilot、CodeLlama等商用/开源开发智能体的表现 |
| **Ours** | 本项目方案 | 经验图谱+全生命周期管理+动态路由的完整方案 |

### 4. 评测指标体系
从4个维度综合评估各方案的性能：
| 维度 | 指标 | 计算方式 | 权重 |
|------|------|----------|------|
| **效果指标** | 任务成功率 | 成功完成的任务数 / 总任务数 | 40% |
| | 平均交互轮次 | 完成单个任务平均需要的用户/系统交互次数 | 15% |
| **效率指标** | 平均任务耗时 | 完成单个任务的平均处理时间 | 15% |
| | 平均token消耗 | 完成单个任务平均消耗的大模型token数 | 10% |
| **稳定性指标** | 错误复现率 | 相同错误再次出现的比例 | 10% |
| | 长周期性能衰减率 | 运行1000个任务后成功率相比初期的下降比例 | 5% |
| **资源指标** | 存储开销 | 经验库/模型的存储空间占用 | 3% |
| | 计算开销 | 完成任务的平均GPU/CPU资源占用 | 2% |

### 5. 实验流程设计
#### 5.1 数据集准备
1. 从公开代码数据集（HumanEval、MBPP、BugNet等）中筛选符合要求的任务样本
2. 补充部分企业实际开发场景中的真实任务样本
3. 对所有任务进行标准化标注，明确预期输出、测试用例、评估标准

#### 5.2 实验环境配置
- 大模型：统一使用Qwen-7B-Chat / CodeLlama-7B作为基础模型，保证对比公平性
- 计算环境：NVIDIA A10 GPU * 1，CPU 16核，内存32GB
- 版本控制：所有实验代码和参数统一版本管理，保证可复现

#### 5.3 实验步骤
1. **冷启动阶段**：所有方案从零开始，无任何历史经验
2. **运行阶段**：各方案依次执行所有评测任务，记录每一步的指标数据
3. **长周期验证**：重复执行任务集3轮，验证经验积累的长期收益
4. **消融实验**：分别禁用经验管理、动态路由等模块，验证各组件的贡献度

#### 5.4 结果分析
1. 对比各方案在不同任务场景下的指标差异
2. 分析经验积累量与性能提升的相关性
3. 验证不同路由策略在不同场景下的适配性
4. 总结方案的优势、不足与适用场景

### 6. 预期实验结果
- 本项目方案在任务成功率上相比Baseline 1提升30%以上，相比Baseline 2提升15%以上
- 平均交互轮次和token消耗相比Baseline 1降低40%以上
- 长周期运行下性能衰减率低于5%，显著优于其他方案
