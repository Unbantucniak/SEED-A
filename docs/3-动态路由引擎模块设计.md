# 动态路由引擎模块设计
## 第三阶段核心交付物

### 1. 模块功能定位
动态路由引擎负责根据当前任务场景、经验库状态与系统资源，自适应选择最优的经验利用方式，实现RAG检索增强与模型微调内化的协同，在保证响应速度的同时提升经验利用效率。

### 2. 经验利用策略空间
路由引擎支持以下4种经验利用策略，覆盖不同场景需求：

| 策略类型 | 适用场景 | 优势 | 劣势 | 资源开销 |
|----------|----------|------|------|----------|
| **RAG即时检索** | 低频、长尾、时效性强的任务 | 响应速度快、无需训练、可解释性强 | 上下文窗口限制、泛化能力弱 | 低 |
| **模板复用** | 高频、标准化、结构固定的任务 | 响应极快、结果稳定 | 灵活性差、无法处理变体任务 | 极低 |
| **提示工程** | 中等频率、有一定变体的任务 | 无需训练、适配性较好 | 依赖高质量prompt、效果不稳定 | 中 |
| **小模型微调** | 高频、高价值、结构相似的批量任务 | 泛化能力强、长期收益高 | 训练成本高、响应延迟高 | 高 |

### 3. 路由决策算法
#### 3.1 特征输入
路由决策基于以下多维度特征综合打分：
1. **任务特征**：任务类型、领域标签、复杂度、历史调用频率、预期收益
2. **经验特征**：匹配经验的数量、平均成功率、平均时效性、相似度分布
3. **系统特征**：当前系统负载、剩余算力、响应时间要求、资源预算

#### 3.2 加权打分模型
每种策略的综合得分计算公式：
```
策略得分 = 预期收益 * 0.4 + 成功概率 * 0.3 + 资源开销评分 * 0.2 + 响应速度评分 * 0.1
```
- 预期收益：使用该策略后相比无经验基线的性能提升预期
- 成功概率：该策略在同类任务上的历史成功率
- 资源开销评分：1/(1+资源开销)，开销越低得分越高
- 响应速度评分：1/(1+预期延迟)，速度越快得分越高

最终选择得分最高的策略执行。

#### 3.3 强化学习动态优化
随着使用数据积累，通过强化学习动态调整各特征的权重，持续优化路由决策准确率：
- 状态空间：当前任务特征、经验库状态、系统状态
- 动作空间：4种经验利用策略
- 奖励函数：任务完成后的实际收益（成功率、节省时间、用户反馈）
- 学习算法：DQN/PPO，离线训练+在线微调

### 4. 渐进式经验内化机制
实现经验从RAG检索到模型内化的平滑迁移：
1. **冷启动阶段**：新经验仅用于RAG检索，收集使用数据
2. **验证阶段**：使用次数超过阈值且成功率>0.8的经验，加入微调候选池
3. **训练阶段**：定期从候选池中选择高价值经验，微调小模型
4. **替换阶段**：微调后效果超过RAG策略的经验，优先使用微调模型处理同类任务
5. **淘汰阶段**：模型内化完成后的经验，可从RAG检索库中下线，减少检索开销

### 5. 模块输出
路由引擎的输出包含：
- 选中的经验利用策略类型
- 匹配到的经验ID列表（RAG/模板/提示策略）
- 微调任务配置（微调策略）
- 预期效果评估与风险提示
