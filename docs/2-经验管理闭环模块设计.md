# 经验管理闭环模块设计
## 第二阶段核心交付物

### 1. 模块功能定位
经验管理闭环模块负责实现经验的全生命周期管理，覆盖"交互数据采集→经验筛选抽取→验证入库→动态更新→淘汰清理"全流程，保障经验库的质量、时效性与紧凑性。

### 2. 核心流程设计
#### 2.1 经验生成与筛选流程
```
原始交互数据 → 数据清洗 → 经验片段抽取 → 质量评估 → 候选经验池 → 验证环节 → 正式入库
```
- **数据采集**：对接智能体交互接口，采集全链路交互数据（用户指令、上下文、操作序列、结果、反馈）
- **数据清洗**：去除无效、重复、敏感信息，标准化数据格式
- **经验抽取**：基于规则+大模型从清洗后的数据中抽取结构化经验单元
- **质量评估**：基于多维指标对候选经验进行打分，低于阈值的直接过滤
- **验证环节**：通过自动化测试/用户确认验证经验有效性，通过后正式入库

#### 2.2 经验更新与演化流程
```
经验调用 → 结果反馈 → 动态元属性更新 → 冲突检测 → 聚类泛化 → 经验库优化
```
- 每次经验被调用后，根据执行结果更新成功率、使用次数等动态元属性
- 定期检测经验之间的冲突，对冲突经验进行人工/自动仲裁
- 定期对高相似性经验进行聚类泛化，合并重复经验，生成更通用的经验

#### 2.3 经验淘汰流程
```
定期扫描 → 失效识别 → 淘汰确认 → 归档/删除
```
- 定期扫描经验库，识别失效经验（时效性<0.2、成功率<0.3、超过1年未使用）
- 淘汰前进行二次确认，避免误删高价值经验
- 淘汰的经验可选择归档到冷存储，或直接删除

### 3. 核心质量评估指标
候选经验进入正式库需要满足以下阈值要求：
| 指标 | 最低阈值 | 权重 |
|------|----------|------|
| 来源可信度 | ≥0.3 | 30% |
| 任务成功率 | ≥0.6 | 40% |
| 通用性 | ≥0.2 | 20% |
| 完整性（必填字段齐全） | 100% | 10% |

综合得分 = 来源可信度*0.3 + 任务成功率*0.4 + 通用性*0.2 + 完整性*0.1 ≥ 0.5 方可入库

### 4. 经验库健康度评价体系
定期对整个经验库的健康度进行评估：
| 指标 | 理想范围 | 说明 |
|------|----------|------|
| 经验平均成功率 | ≥0.8 | 整体经验质量越高越好 |
| 经验平均时效性 | ≥0.7 | 经验整体越新越好 |
| 冗余率 | ≤20% | 相似重复经验占比越低越好 |
| 冲突率 | ≤5% | 存在冲突的经验占比越低越好 |
| 调用覆盖率 | ≥60% | 有被调用过的经验占比越高越好 |

健康度得分 = 平均成功率*0.4 + 平均时效性*0.2 + (1-冗余率)*0.2 + (1-冲突率)*0.1 + 调用覆盖率*0.1
